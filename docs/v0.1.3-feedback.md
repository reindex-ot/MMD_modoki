## 詳細

### V013-001 ボーンリストを選択中、クリック位置と異なるボーンが選択される
- 種類: bug
- 優先度: P1
- 発生日: 2026-03-19
- 送信元: X のリプライ
- 状態: fixed
- 影響範囲: ボーン選択UI, タイムライン, ボーンオーバーレイ
- 現象: リストを上から順に選んでいくと、ある箇所で数行下の別ボーンが選択される
- 対応内容:
  - ボーン欄の `bone-select` を通常のドロップダウンに戻した
  - タイムラインのスクロール時に選択がずれないよう、左右のスクロール範囲を揃えた
  - タイムラインのクリック選択で `scrollTop` を二重加算しないよう修正した
  - ボーン欄、タイムライン、ボーンオーバーレイの選択同期を整理した
- 確認済み:
  - Windows の拡大率 100% でも再現
  - 3 種類のモデルで再現
  - 特定モデル依存ではない

### V013-009 カメラのキーフレーム登録が不正になる
- 種類: bug
- 優先度: P0
- 発生日: 2026-03-19
- 送信元: GitHub issue
- 状態: fixed
- 影響範囲: カメラのキー登録, source animation の同期, 編集時の初期化, 停止中プレビュー
- 対応内容:
  - `track.positions` を viewport camera position ではなく `camera target` として保存するよう統一
  - `track.distances` は負値、`track.fovs` は degree で扱うよう揃え、`MmdCamera` runtime と editor の意味ずれを解消
  - 停止中は `mmdCamera -> viewport` の毎フレーム同期を止め、frame move 時だけ sampled pose を再適用する形に整理
  - play 開始時は runtime handle 再生成後に current frame へ再 seek し、close-up や左右反転が出ないよう修正
- 確認済み:
  - register 直後に white out / close-up しない
  - frame move 後に左右反転しない
  - 再生開始時に close-up せず camera motion を継続再生できる

### V013-011 半透明とスフィア表示が崩れる
- 種類: bug
- 優先度: P1
- 発生日: 2026-03-19
- 送信元: GitHub issue
- 状態: investigating
- メモ:
  - `.x` アクセサリ側は `png / bmp / tga / webp` の拡散テクスチャに `hasAlpha / useAlphaFromDiffuseTexture` を立てる補助対応を入れた
  - PMX 側も透過評価の取りこぼしに備えて、拡散テクスチャの alpha 利用フラグを補完するフォールバックを入れた
  - cutout 寄りの暫定回避として `AlphaCutOff` シェーダープリセットを追加した
  - `Luminous` は `GlowLayer` の深度無視による貫通を避けるため、Bloom ベースの発光処理へ変更した
  - 顔だけ横縞になるモデルで再確認したが、現時点では改善なし
  - `Unlit Flat` でも縞が残るため shader より `depth / z-fighting` 系の疑いが強い
  - sphere / texture 読み込み側は引き続き切り分け中

### V013-012 BMP / PNG の白黒アルファチャンネルが反映されない可能性がある
- 種類: bug
- 優先度: P1
- 発生日: 2026-03-24
- 送信元: X のポスト
- 状態: investigating
- 影響範囲: テクスチャ読み込み, 透過表現, 材質表示
- 現象:
  - 白アルファ / 黒アルファのチャンネルを使った透過表現が反映されていないように見える
  - 投稿者メモでは `ボーン○ / 物理○ / 表示△` のうち表示面の問題として報告
- メモ:
  - BMP / PNG のアルファチャンネル解釈不足、または MMD 側の透過マスク表現との差異の可能性
  - `V013-011` の sphere / texture 読み込み系と同根か切り分けが必要
  - コード確認では PMX 側は `babylon-mmd` loader が透過評価を持っている一方、`.x` アクセサリ側は拡散テクスチャに `hasAlpha / useAlphaFromDiffuseTexture` を立てておらず、まずこちらを修正対象とした
  - PMX 側の追加フォールバックは入れたが、顔の縞問題とは別件の可能性が高い
  - `stage01.x` の再現確認では `.x` 内で `screen.bmp` を参照している一方、実ファイルが配布物から欠けていた
  - そのため `screen.bmp` 未反映の件はローダー不具合ではなく素材欠品の可能性が高い
  - `.x` テクスチャ探索自体は近傍ファイル探索のフォールバックを追加済み
  - 当面はこの件を優先して継続調査する
