### FB-004 WebGPU が動いていない環境が多い
- 日付: 2026-03-10
- 種別: 不具合
- 分類: 描画 / 環境依存
- 状態: 確認中
- メモ:
  - OS, GPU, ドライバ, Electron 既定挙動の影響が大きい。

  - `WebGPU (WGSL)` 表示でも一部機能が fallback / disable の可能性がある。

### FB-005 モデルによって顔の描画が壊れる
- 日付: 2026-03-10
- 種別: 不具合
- 分類: 描画 / モデル互換
- 状態: 確認中
- メモ:
  - 描画順、透明材質、裏面描画、`alpha` 補正、face layer の競合を疑う。
  - `Unlit Flat` でも崩れるモデルがあり、陰影計算より `z-fighting / depth競合` の疑いが強い。
  - `docs/face-render-corruption-investigation.md` を参照。

### FB-006 Ubuntu 24.04 で動作しない報告
- 日付: 2026-03-10
- 種別: 不具合
- 分類: 起動 / Linux
- 状態: 確認中
- メモ:
  - コンソール内容から `chrome-sandbox` の SUID 設定不備で abort している可能性が高い。
  - Ubuntu 24.04 固有というより Linux zip 配布時の Electron/Chromium サンドボックス要件の問題と考えられる。
  - `--no-sandbox` 指定で起動できた報告あり。
  - packaged build では Linux 限定で `--no-sandbox` / `--disable-setuid-sandbox` を付ける暫定対応を入れた。
  - 恒久対応は AppImage / Flatpak などの配布形式見直しで対応したい。

## FB-005 調査ログ
- `Unlit Flat` でも顔の縞が消えず、shader より `depth` と重ね面の競合を優先して疑う段階。
- 試した軽量対策:
  - `backFaceCulling` 一律強制の停止
  - 透明材質向け `alpha` 補正の抑制
  - `babylon-mmd` 材質描画モードの `AlphaEvaluation` 化
  - 透明寄り材質への軽い `zOffset` と `alphaIndex` 付与
- 結果:
  - 報告モデルでは改善確認できず。
  - 次版では無理に修正を入れず、再現モデル前提での切り分けを優先する。
