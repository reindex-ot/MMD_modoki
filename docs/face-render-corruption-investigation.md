# PMXモデル顔崩れ調査メモ

更新日: 2026-03-10

対象:
- `src/mmd-manager.ts`
- PMX/PMD モデル表示
- WebGL2 / WebGPU 共通の描画問題

## 概要
- 一部 PMX モデルで顔の描画が崩れる報告がある。
- 症状は 2 系統ある。
  - 透明材質や裏面描画が絡む前後関係の破綻
  - 顔表面に横縞が出る `z-fighting` 系の破綻
- 特に `Unlit Flat` でも横縞が残るモデルは、陰影計算より `depth` と重ね面の競合を優先して疑うべき。

## 主な原因候補

### 1. 描画順
- 顔、まつげ、口内、前髪などの透明/半透明材質は描画順の影響を強く受ける。
- PMX は材質配列順と `indexCount` で描画範囲を持っており、順序自体は存在する。
- ただし Babylon 側の透明ソートや depth 処理で、その前後関係が崩れる可能性がある。

### 2. 裏面描画
- `backFaceCulling = false` を一律で強制すると、顔の内側や口内、重なり面が見えやすくなる。
- 片面前提のモデルでは特に悪化要因になる。

### 3. `alpha` 補正
- 読込後に `alpha === 0` の材質を補正している箇所がある。
- 透明材質まで不必要に触ると、まつげや顔のデカール面が不正に前へ出る可能性がある。

### 4. 透明材質の depth / shadow
- 透明材質が depth に強く書き込むと、顔に穴が開く、前後関係が壊れる、縞が見えるなどの破綻が起こる。
- 顔の重ね面が極端に近いモデルでは `z-fighting` に見えることがある。

### 5. Shader 差分
- `toon`、`sphere`、`shadow` 差分で崩れて見えるケースはある。
- ただし `Unlit Flat` でも崩れる場合は本命ではない。

## 症状別の見分け方
- 目元やまつげの抜けがおかしい:
  - 透明材質、描画順、`alpha` 補正
- 顔の内側や口内が見える:
  - 裏面描画
- 顔だけ横縞になる:
  - 重ね面の `z-fighting`
- 光り方だけ不自然:
  - shader / toon / sphere

## 2026-03-10 時点で試した軽量対策
- `backFaceCulling` 一律強制の停止
- 透明材質向け `alpha` 補正の抑制
- `babylon-mmd` 材質描画モードの `AlphaEvaluation` 化
- 透明寄り材質への軽い `zOffset` と `alphaIndex` 付与

結果:
- 報告モデルでは改善確認できず。
- `Unlit Flat` でも顔だけ横縞が残るため、陰影計算より `z-fighting / depth競合` の疑いが強い。

## 現時点の優先度
本命:
- 顔の重ね面による `z-fighting`
- 透明材質の depth
- 描画順

次点:
- 裏面描画
- `alpha` 補正

後順位:
- toon shader 差し替え
- WebGPU 固有差分
- morph / deformation

## 次にやるなら
- 再現モデルを固定して、顔材質と重ね面の実体を確認する
- 顔まわりの材質だけ個別に `depth write`, `alpha blend/test`, `zOffset` を比較する
- 必要なら材質単位の診断表示を追加する

## 次版の扱い
- 軽量対策で改善しないため、次版での無理な修正は見送る方が安全
- 既知不具合として保持し、再現モデル前提で調査継続とする
